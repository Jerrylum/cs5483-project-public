# Optuna Dashboard for CS5483 Project

This directory contains the Optuna dashboard setup for the CS5483 project. The dashboard visualizes optimization results stored in a SQLite database generated by Optuna experiments.

## Features

- Optuna Dashboard web interface
- Direct connection to existing SQLite database
- Containerized with Docker for easy deployment
- Custom Docker image with pre-installed dependencies for fast startup
- Port 8080 exposed for web access

## Prerequisites

- Docker and Docker Compose installed on your system
- SQLite database with Optuna study results at `../../data/result.sqlite3`

## Setup Instructions

1. **Build and start the Optuna Dashboard container**
   ```bash
   cd apps/optuna-dashboard
   docker-compose up -d --build
   ```

   The first run will build the custom Docker image with all dependencies pre-installed.

2. **Access the dashboard**
   
   Open your browser and navigate to:
   ```
   http://localhost:8080
   ```
   
   Or if accessing remotely, replace localhost with your server IP.

3. **Verify the container is running**
   ```bash
   docker-compose ps
   ```

4. **Check the logs**
   ```bash
   docker-compose logs
   ```

## Managing the Dashboard

### Starting and Stopping

```bash
# Start the dashboard
docker-compose up -d

# Stop the dashboard
docker-compose down

# Rebuild the image if Dockerfile changes
docker-compose up -d --build
```

### Viewing Logs

```bash
# View logs
docker-compose logs

# Follow logs
docker-compose logs -f
```

## Connecting to a Different SQLite Database

If you need to connect to a different SQLite database, modify the volume mapping in `docker-compose.yml`:

```yaml
volumes:
  - /path/to/your/database.sqlite3:/app/result.sqlite3:ro
```

## Troubleshooting

- **Container fails to start**: Check the logs with `docker-compose logs optuna-dashboard`
- **Cannot access dashboard**: Ensure port 8080 is not already in use by another application
- **Database connection issues**: Verify the SQLite database exists at the specified path
